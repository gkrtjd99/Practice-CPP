# Chapter04 Computation

## 복습문제

### 계산이란 무엇인가?
**계산**이란 주어진 입력을 바탕으로 출력을 생성하는 과정이다.
### 계산의 입력과 출력은 어떤 의미인가? 예를 들어보자.
**입력**은 키보드, 마우스, 터치스크린, 파일, 다른 입력 장치, 다른 프로그램, 또는 프로그램의 다른 부분에서 올 수 있는 데이터를 의미한다.
**출력**은 화면, 파일, 네트워크 연결, 다른 출력 장치, 다른 프로그램, 또는 프로그램의 다른 부분으로 가는 데이터를 의미한다.
예를 들어, 달력 프로그램의 경우, 주요 입력은 월과 일을 보거나 약속을 기록하도록 요청하는 것이다. 출력은 달력과 약속의 표시, 그리고 화면에 나타나는 입력용 버튼과 프롬프트다.

### 표현식은 어떤 일을 하는가?
표현식은 피연산자의 값으로부터 결과값을 계산한다.
### 4장에서 설명한 구문과 표현식의 차이는 무엇인가?
**표현식(expression)**은 피연산자와 연산자를 사용하여 값을 계산하는 반면 , **구문(statement)**은 프로그램의 동작을 지정하는 언어 구성 요소이다.
### `lvalue`란 무엇인가? `lvalue`를 필요로 하는 연산자의 예를 들어보자. 왜 어떤 연산자엔 `lvalue`가 필요하고 다른 연산자는 그렇지 않은가?
`lvalue`는 대입문의 왼쪽에 올 수 있는 객체를 가리키는 표현식, 즉 값을 대입할 수 있는 모든 표현식을 말한다.
lvalue를 필요로 하는 연산자의 예시
- 선증가/감소 연산자 : `++lval`, `--lval`
- 대입 연산자 : `lval = a`
- 복합 대입 연산자 : `lval += a`, `lval *= a` 등
`lvalue`가 필요한 연산자들의 값을 변경하거나 피연산자의 메모리 위치를 참조해야 하기 때문이다.
### 상수 표현식은 무엇인가?
컴파일 시점에 값이 평가될 수 있는 표현식이다.
`const`, `constexpr`을 사용하여 상수로 표현한다면 초기화 이후 값을 변경할 수 없게 되고 컴파일 시점에 값이 정해져 있게 된다.
### 리터럴은 무엇인가?
리터럴(literal)은 값을 직접적으로 지정하는 표기법이다.
### 표기 상수란 무엇이고, 왜 사용하는가?
표기 상수(symbolic constant)는 컴파일 시에 값이 알려지지만, 초기화 이후에는 새로운 값을 할당할 수 없는 이름이 있는 객체다.
`constexpr` 또는 `const` 키워드를 사용하여 선언된다.
표기 상수를 사용하는 이유
- 코드 가독성 도움 : `3.14159`와 같은 직접적인 숫자 대신 `pi`와 같은 의미있는 이름을 사용하면 가독성이 올라간다.
### 마법의 상수란 무엇인가? 예를 들어보자.
기호 상수를 정의하는 부분을 제외한 코드에서 사용된 명확하지 않은 리터럴을 **마법의 상수**라 부르기도 한다.
```
constexpr int max = 17; // 리터럴은 상수 표현식임
int val = 10;
max+2; // 상수표현식
val+2; // 상수표현식이 아님 : 변수를 사용함
```
### 정수와 부동소수점 수에 사용할 수 있는 여산자에는 무엇이 있는가?
정수 `int`와 부동소수점 수`double` 모두에 사용할 수 있는 공통적인 연산자
- 대입(assignment): `=`
- 덧셈(addition): `+` 
- 뺄셈(subtraction): `-` 
- 곱셈(multiplication): `*` 
- 나눗셈(division): `/` 
- 증가(increment by 1): `++`
- 감소(decrement by 1): `--` 
- 복합 대입(compound assignment): `+=`, `-=`, `*=`, `/=` 
- 읽기/쓰기: s `>>` x (입력), s `<<` x (출력) 
- 비교(comparison): `==`, `!=`, `<`, `<=`, `>`, `>=` 
### 정수에는 사용할 수 있지만 부동소수점 수에 사용할 수 없는 연산자는 무엇인가?
나머지 연산자 ```는 정수(int)에만 사용할 수 있으며 부동소수점 수(double)에는 직접 사용할 수 없다.
### `string`에 사용할 수 있는 연산자에는 무엇이 있는가?
- 대입(assignment): `=`
- 연결(concatenation): `+` 
- 복합 대입(append): `+=`
- 읽기/쓰기: s `>>` x (입력), s `<<` x (출력) 
- 비교(comparison): `==`, `!=`, `<`, `<=`, `>`, `>=`
### 프로그래머는 어떨 때 `if` 구문보다 `switch` 구문을 선호하는가?
중첩된 `if`구문보다 간단하여 비교할 상수가 여러개인 경우 선호한다.
### `switch` 구문의 일반적인 문제점은 무엇인가?
- `break`누락: 각 case 끝에 break를 누락하는 것이 가장 흔하게 발생. 의도치 않게 다음 case로 실행 흐름이 이어지는 오류가 발생할 수 있다.
- `default`누락 : 모든 대안을 완벽하게 나열했다 자신하지 못하면 `default`레이블을 작성하는 것이 좋다. 예상하지 못한 case가 있을 수 있음.
- `string` 불가 : `switch`문은 상수값을 비교하는데 최적화 되어 있다.
- 상수 표현식만 허용 : `case`레이블의 값은 반드시 서로 다른 상수여야 한다.
### `for` 루프에서 헤더라인의 각 부분은 각각 어떤 기능을 하며, 어떤 순서로 실행되는가?
- 초기화(initialization): 루프 변수를 초기화한다 (예: int i = 0;). 이 부분은 루프가 시작될 때 한 번만 실행된다.
- 조건(condition): 루프를 계속 실행할지 여부를 결정하는 조건이다 (예: i < 100;). 이 조건은 루프 본문이 실행되기 전에 매번 평가된다. 조건이 true이면 루프 본문이 실행되고, false이면 루프가 종료된다.
- 증가(increment): 루프 변수의 값을 변경한다. (예: ++i;). 이 부분은 루프 본문이 매번 실행된 후에 실행된다.
### `char`의 숫자 값은 어떻게 출력하는가?
`char` 변수의 숫자 값(아스키/유니코드 값)을 출력하려면 해당 char를 int로 **명시적 변환**해야 한다.
```
cout << int(ch) << '\n'; //아스키 코드 출력
cout << ch; //문자출력
```
### 함수 정의에서 `char foo (int x)`의 의미를 설명하라
- `char` : 이 함수가 `char` 타입의 값을 **반환**한다는 것을 나타낸다.
- `foo` : 함수의 **이름**이다.
- `(int x)`: 이 함수가 `int` 타입의 **인자**를 `x`라는 이름으로 **매개변수**로 받는다는 것을 나타낸다.
### 어떤 경우에 프로그램의 한 부분을 분리된 함수로 정의해야 하는가? 그 이유를 들어보라.
- 계산과정을 논리적으로 구별한다.
- 프로그램 텍스트를 깔끔하게 한다.
- 한 함수를 프로그램 안에서 여러 번 사용한다.
- 테스트를 쉽게 한다.
### `int`로 할 수 있는 일 중에 `string`으로 할 수 없는 일은 무엇인가?
- 나머지 연산 : `%`연산은 `int`에만 적용된다.
- 정수 산술 연산 : `int`는 `string`에 정의되지 않은 사친연산 이외의 다른 정수 산술연산이 가능하다.(예:비트연산)
- 수학적 계산 : `sqrt()`, `log()`, `exp()` 등과 같은 표준 수학 함수는 int 타입의 숫자 값을 인수로 받거나 int로 변환될 수 있는 숫자 값을 처리한다. 
`string`은 이러한 함수에 직접 사용될 수 없다.
### `string`으로 할 수 있는 일 중에 `int`로 할 수 없는 일은 무엇인가?
- 문자열 연결 : `+` 연산자를 사용하여 문자열을 연결하는 것은 string에만 해당된다.
- 텍스트 조작 : 특정 문자열 내 검색, 특정위치 문자열 삽입 등 텍스트 기반 조작은 `string`타입에서만 가능하다.
- 문자열 기반 비교 : 사전식순서에 따른 문자열 비교는 `string`에만 해당된다.
### `vector`의 세번째 요소의 인덱스는 몇인가?
2
### `vector`의 모든 요소를 출력하는 `for`루프는 어떻게 작성하는가?
```
vector<int> v = {5, 7, 9, 4, 6, 8}; 
for (int x : v)
    cout << x << '\n';
```
### `vector<char> alphabet (26);`은 어떤 일을 수행하는가?
`char`타입 요소 26개의 공간을 가지는 alphabet이란 `vector`생성한다.
### `push_back()`은 `vector`에서 무슨 일을 하는가?
push_back()은 vector의 **끝(back)**에 새로운 요소를 추가한다.
### `vector`의 멤버 `size()`는 무슨일을 하는가?
vector의 멤버 함수 size()는 vector에 포함된 요소의 개수를 반환한다.
### `vector`가 널리 쓰이고 유용한 이유는?
- 크기 조절의 유연성: vector는 미리 크기를 지정할 필요가 없으며, push_back()과 같은 연산을 통해 원하는 만큼 요소를 추가할 수 있다.
- 효율적인 요소 접근: 인덱스(첨자)를 통해 요소에 효율적으로 접근할 수 있다.
- 메모리 관리의 단순성: vector는 내부적으로 메모리 할당 및 해제를 관리하여 프로그래머가 저수준의 메모리 관리에 대해 걱정할 필요가 없도록 한다.
- 표준 라이브러리 지원: C++ 표준 라이브러리의 일부로서 `sort()`, `find()` 등 다양한 표준 알고리즘과 함께 사용할 수 있다.
### `vector`의 요소를 어떻게 결정하는가?
vector의 요소는 vector를 선언하거나 초기화할 때 템플릿 인자를 통해 지정된다.
- `vector<double>`은 `double` 타입의 요소를 가지는 `vector`를 의미
- `vector<string>`은 `string` 타입의 요소를 가지는 `vector`를 의미
